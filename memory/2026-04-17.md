## 做错的事情（What Went Wrong）

1. **被动响应系统崩溃**：exec 进程被 SIGKILL 批量杀掉（mild-roo、sharp-sh、lucky-fa、plaid-sa、tide-sag、mellow-c、kind-wil、tidy-riv 等），用户多次问"好了没？""好了通知我"，OpenClaw 没有主动告知状态，导致用户反复 ping。这说明异步任务失败时没有主动上报机制。

2. **微信验证码自动化失败**：花了大量时间尝试用 Playwright 自动过验证码（drag slider、iframe 检测），最终失败。这段时间消耗了用户大量耐心。

3. **Gemini CLI 安装白费**：认证成功但 API 权限被 403，用户最后说"你帮我删掉gemini cli吧，用不了，好难受"。在确认可用性之前就花时间装好，是浪费。

4. **重复检查**：用户说"你再检查一遍"和"重新检查一遍，要按最新数据来检查"，说明第一次检查不够完整或没给出明确结论。

5. **系统 exec 稳定性差**：大量 exec 被 SIGKILL，可能与 WSL 资源限制或进程管理有关，没有提前预防。

## 改进点（Improvements）

1. **异步任务失败主动上报**：当 exec 或子 agent 失败时，应该主动发消息通知用户，而不是等用户来问。当前依赖用户轮询，效率低且体验差。

2. **系统健康检查脚本**：建立一个快速诊断脚本，检查：npm 包完整性、exec 进程状态、Chrome/Playwright 可用性、MCP server 连接性。在用户要求"全身检查"时能一键输出结构化报告。

3. **外部工具安装前先验证可行性**：Gemini CLI 安装前先检查账号是否有 Code Assist 权限，避免装完不能用。微信文章读取前先测试普通链接是否能访问，再指导用户操作。

4. **微信验证码不自动化**：明确告知用户微信验证码无法自动绕过（可能涉及人机验证），避免无效尝试。可以建议用户手动在浏览器验证后复制内容。

5. **长任务分阶段确认**：对于需要多步骤的操作（如安装 Gemini CLI），分阶段报告进展（安装中/认证中/测试中），不要等到最后才发现失败。

6. **SIGKILL 问题根因排查**：记录哪些命令/场景容易触发 SIGKILL，排查 WSL 资源限制或 cgroup 设置，给出预防建议。

### 新增知识条目

**微信文章读取最佳实践**
- 微信文章不能直接分享（会带微信登录态，触发 wappoc_appmsgcaptcha 验证码）
- 正确方式：用"分享链接"（mp.weixin.qq.com/s/xxx?sharer_shareinfo=...）方式，OpenClaw Playwright 干净 UA 可以直接读取，无需验证码
- 微信小露的 web_fetch 走不通，需要调用 Playwright
- Windows Chrome（有微信登录态）会被拦，Linux Playwright（干净 UA）不会被拦

**Gemini CLI 权限问题**
- Google 账号 + Gemini Code Assist 授权后仍可能 403 PERMISSION_DENIED
- 可能原因：账号无 Code Assist 权限或地区限制
- 目前不可用，安装需谨慎评估

**系统 SIGKILL 问题**
- WSL 环境下 exec 进程会被 SIGKILL 批量杀掉
- 触发场景：长时间运行的 node 脚本、headed 模式 Playwright、npm install
- 建议：任务加 timeout、headless 优先、及时回收进程
